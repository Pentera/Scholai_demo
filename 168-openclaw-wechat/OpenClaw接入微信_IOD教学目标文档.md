# 教学目标文档（Instructional Objectives Doc）

## 课程信息

| 字段 | 内容 |
|------|------|
| **课程名称** | OpenClaw 接入微信——官方 ClawBot 插件安装与使用教程 |
| **课程简介** | 本课程由小林（AI学栈栏频道）讲解如何通过微信官方推出的 ClawBot 插件，将 OpenClaw AI 助手接入个人微信。内容涵盖前提条件（微信版本更新）、一行命令行安装 ClawBot 插件、扫码绑定微信账号，以及在微信聊天窗口直接调用 AI 获取资讯的完整演示。 |

### 学习者先备能力要求

| 维度 | 内容 |
|------|------|
| **目标学习者** | 已在使用或有意使用 OpenClaw 的个人用户；具备基本智能手机操作能力 |
| **学科前置知识** | 了解 OpenClaw/AI 助手的基本概念；了解微信的基本操作（设置、插件入口） |
| **学科/工具能力** | 能够打开本地终端（命令行）并执行基础命令（如 `npx -y <命令>`）；拥有微信账号并能扫码登录 |

### EX-01：微信升级并定位 ClawBot 插件入口

| 字段 | 内容 |
|------|------|
| **示例 ID** | EX-01 |
| **来源位置** | 视频开始约 0:40 - 1:30（介绍安装前置步骤段落） |
| **示例类型** | 方法示范 |

#### 题目原文
在安装之前，先打开微信，把它升级到最新版本。然后在右下角点击"我"→"设置"→"插件"，查看是否显示微信 ClawBot 插件。若更新后仍未出现，在手机后台将微信划掉，再重新进入微信，基本上就可以看到了（目前处于灰度状态）。

#### 求解目标
确认微信已具备 ClawBot 插件入口，满足后续安装前提条件。

#### 解题步骤
1. 打开应用商店，将微信更新至最新版本
2. 进入微信 → 右下角"我" → "设置" → "插件"
3. 检查列表中是否出现"微信 ClawBot"
4. 若未出现：从手机后台彻底关闭微信进程，重新打开微信
5. 再次进入"插件"查看，通常即可看到 ClawBot 入口

#### 正确答案
插件列表中出现"微信 ClawBot"即为成功

#### 结构化条件
| 条件编号 | 条件内容 | 条件类型 |
|----------|----------|----------|
| C1 | 微信已安装在手机上 | 初始状态 |
| C2 | 微信 ClawBot 处于灰度推送阶段 | 约束条件 |
| C3 | 部分用户不会第一时间收到推送 | 约束条件 |

#### 学科结构（用于生成变式题）
| 维度 | 描述 |
|------|------|
| **核心逻辑关系** | 版本更新 → 功能解锁（灰度推送）→ 重启触发推送检测 |
| **关键操作类型** | 应用更新 + 功能入口检查 + 进程重启触发灰度刷新 |
| **可变参数** | 具体 App 名称（微信→其他 IM）、功能名称（ClawBot→其他插件） |
| **结构不变量** | "更新→检查入口→未见则重启→再检查"的灰度功能激活流程 |

#### 题型特征标签
`#应用更新` `#灰度功能激活` `#前置环境检查` `#IM插件`

### EX-02：命令行安装 ClawBot 并扫码绑定

| 字段 | 内容 |
|------|------|
| **示例 ID** | EX-02 |
| **来源位置** | 视频约 1:30 - 2:30（正式安装步骤演示段落） |
| **示例类型** | 方法示范 |

#### 题目原文
打开本地终端（如 Google 的 antgularity 终端），输入 `npx -y` 加上官方提供的安装指令（安装官方版本的微信 OpenClaw 对话插件），回车执行。安装完成后，终端会弹出一个二维码，用要绑定的微信号扫码，手机端点击"连接"完成绑定。

#### 求解目标
完成 ClawBot 插件安装并将其绑定到指定微信账号。

#### 解题步骤
1. 打开本地终端（macOS Terminal / Windows PowerShell 等）
2. 输入官方安装命令：`npx -y <官方ClawBot安装包名>`，回车
3. 等待安装完成，终端输出二维码
4. 打开目标微信号，扫描终端中的二维码
5. 手机端弹出"是否将 OpenClaw 连接到微信"提示，点击"连接"
6. 手机进入 ClawBot 对话窗口，绑定完成

#### 正确答案
手机微信出现 ClawBot 对话窗口，且 AI 可正常响应消息

#### 结构化条件
| 条件编号 | 条件内容 | 条件类型 |
|----------|----------|----------|
| C1 | 本地已安装 Node.js 运行环境（支持 npx） | 初始状态 |
| C2 | 官方 ClawBot 安装包已发布到 npm | 初始状态 |
| C3 | 微信账号已完成 Step 0（插件入口存在） | 约束条件 |
| C4 | 绑定的微信号即为使用 ClawBot 的账号 | 过程变化 |

#### 学科结构（用于生成变式题）
| 维度 | 描述 |
|------|------|
| **核心逻辑关系** | 命令行安装工具 → 生成授权凭证（二维码） → 扫码完成账号绑定 |
| **关键操作类型** | npx 包安装 + 二维码授权 + IM 账号绑定 |
| **可变参数** | 命令名称、二维码形式（也可能是 Token）、目标平台（微信→其他 IM） |
| **结构不变量** | "安装CLI工具 → 生成授权码 → 扫码/粘贴授权 → 确认绑定"这一 OAuth-like 流程 |

### EX-04：ClawBot 限制说明与 Agent 切换演示

| 字段 | 内容 |
|------|------|
| **示例 ID** | EX-04 |
| **来源位置** | 视频约 3:20 - 4:16（使用限制与总结段落） |
| **示例类型** | 概念演示 |

#### 题目原文
当前绑定的是默认的主 Agent。若要切换绑定的 Agent，可通过 `openclaw config` 命令将账号绑定到指定的 Agent。此外：①一个微信只能绑定一个 OpenClaw，如需更换账号需重新扫码；②目前插件只支持手机端，电脑版微信即使更新也没有 ClawBot 入口；③官方插件保障安全性：他人无法添加你的 ClawBot，也无法将其拉入群聊；④目前只支持个人单点使用。

#### 求解目标
理解 ClawBot 的使用边界，避免误操作；了解 Agent 切换方法。

#### 解题步骤
1. （切换 Agent）在终端执行 `openclaw config`，将账号绑定到目标 Agent
2. （重新绑定）若需更换绑定的 OpenClaw 账号，重新执行安装流程并扫码
3. （确认设备限制）仅在手机版微信中使用，电脑版不支持

#### 正确答案
- 成功切换 Agent：ClawBot 对话窗口中 AI 身份切换为目标 Agent
- 使用限制记忆正确

#### 结构化条件
| 条件编号 | 条件内容 | 条件类型 |
|----------|----------|----------|
| C1 | 已完成初始绑定（EX-02 已完成） | 初始状态 |
| C2 | 一个微信只能绑定一个 OpenClaw 实例 | 约束条件 |
| C3 | 手机端微信支持 ClawBot，电脑端暂不支持 | 约束条件 |
| C4 | 官方插件隔离性：他人无法访问你的 ClawBot | 约束条件 |

#### 学科结构（用于生成变式题）
| 维度 | 描述 |
|------|------|
| **核心逻辑关系** | 绑定约束（单点）+ 设备约束（手机端）+ 安全隔离（私有）+ 可配置性（Agent切换） |
| **关键操作类型** | 配置命令执行 + 使用约束判断 |
| **可变参数** | 具体命令名称、平台名称（微信→其他 IM） |
