第一部分 › 先发优势

第一部分：为什么这次更新很重要——AI 嵌入 IM 的先发优势

微信官方正式推出了 OpenClaw 插件，名称叫做 ClawBot。这件事情的意义非常大：它意味着 OpenClaw 终于可以更安全、更方便地接入个人微信，你不再需要来回切换 App，也不需要折腾复杂的第三方方案。装完之后，你可以在自己的微信里直接和 OpenClaw 进行对话——就像和好友聊天一样，随时调用自己的 AI Agent。

这一集成模式的核心逻辑是：不改变现有使用习惯 → 降低使用门槛 → 先发优势越大。随着越来越多的平台接入 OpenClaw，AI 嵌入 IM 很可能会成为人们与 Agent 交互的主流方式。

需要注意的是："ClawBot 并非第三方插件或民间魔改版"，所以无需担心账号安全和隐私问题。他人无法添加你的 ClawBot，也无法将其拉进群聊。

可以这样理解："把 AI 助手直接塞到微信里"，就是零切换、即用即得的最佳注脚。

第二部分 › 安装前准备

第二部分：安装前的准备——微信升级与插件入口定位

在开始安装之前，我们需要先完成一个前置步骤：确认微信已升级到最新版本，并找到 ClawBot 插件的入口。

操作步骤：

  1. 打开应用商店，将微信更新至最新版本
  2. 进入微信 → 右下角点击"我" → "设置" → "插件"
  3. 检查列表中是否出现"微信 ClawBot"

目前 ClawBot 处于灰度推送阶段，并非所有用户都会在第一时间收到推送。如果更新后在插件列表中没有看到 ClawBot，不用担心——记住："看到更新后找不到新功能入口，立刻想到后台划掉 App 进程，重启触发灰度推送检测"。

解决方法：
从手机后台彻底关闭微信进程，重新打开微信后再次进入"插件"界面，通常就能看到 ClawBot 入口了。

需要注意的是："不要在没有看到 ClawBot 入口的情况下就继续安装步骤"，正确做法是先确认入口可见，再进行后续操作。

第二部分 › 操作示例
【操作示例】定位 ClawBot 插件入口

情境：打开微信后更新到最新版本，但在"设置→插件"中没有看到 ClawBot。

操作步骤：

  1. 确认微信已更新至最新版本（应用商店中显示"已是最新"）
  2. 进入微信 → "我" → "设置" → "插件"
  3. 若未出现 ClawBot：从手机后台彻底关闭微信
  4. 重新打开微信，再次进入"设置→插件"
  5. 查看 ClawBot 入口是否出现

结果：插件列表中出现"微信 ClawBot"即为成功，可进行下一步安装。

第三部分 › 安装与绑定

第三部分：核心安装流程——命令行一键安装与扫码绑定

一、命令行安装 ClawBot

安装过程非常简单。官方提供了一行命令行指令，让整个安装过程像"一键安装 App"一样便捷。

打开本地终端（macOS 的 Terminal、Windows 的 PowerShell 等均可），输入官方提供的安装命令：

npx -y <官方ClawBot安装包名>

其中，npx -y 是 Node.js 包管理工具的自动确认参数，意思是"自动安装并运行，无需手动确认权限"；后面跟上官方 ClawBot 安装包名称，回车执行即可。

简单来说："看到 npx -y，立刻想到使用 Node 包管理工具自动安装，不需要手动确认权限"。

需要注意的是："不要自行搜索安装命令，应使用官方文档或教程中提供的完整命令"，确保安装的是官方版本。

二、扫码绑定微信账号

命令执行完毕后，终端会自动弹出一个二维码。

此时：

  • 打开你要绑定的微信号
  • 扫描终端中的二维码
  • 手机端弹出确认提示"是否将 OpenClaw 连接到微信"，点击"连接"

需要特别注意："用哪个微信号扫码，就是将 ClawBot 绑定到哪个账号"。后续使用 ClawBot 必须在该账号下进行，请提前确认好要绑定的账号。

记住："看到终端弹出二维码，立刻想到选对微信号 → 扫码 → 手机端点击'连接'"。

第三部分 › 操作示例
【操作示例】命令行安装 ClawBot 并完成扫码绑定

前置条件：已找到 ClawBot 插件入口；本地已安装 Node.js（支持 npx）

操作步骤：

  1. 打开本地终端

结果：手机微信出现 ClawBot 对话窗口，绑定完成。

第四部分 › 效果验证

第四部分：效果验证——在微信里和 AI 对话

绑定完成后，打开手机微信中的 ClawBot 对话窗口，开始验证。

第一步：验证连通性

发送"你好"。

如果绑定成功，AI 会立刻回复（示例回复："你好，我是啪啪虾。"）。

第二步：验证任务能力

发送一个任务型请求，例如："帮我查询24小时最新AI资讯"。

AI 会返回结构化的资讯摘要，例如：

  • 乐鲲成立 AMILabs，融资十亿
  • 安士博完成300亿美元融资

这说明 ClawBot 不只是一个聊天入口，而是能够调用 OpenClaw Agent 执行真实任务的接口。

可以这样理解："先发'你好'测连通 → 再发任务测能力"，就像装好设备后"先开机看看亮不亮，再运行程序看功能"。

第五部分 › 四项限制

第五部分：使用边界——你需要知道的四项限制

在正式使用 ClawBot 之前，有四项核心限制需要了解：

限制一：单点绑定

一个微信账号只能绑定一个 OpenClaw。如果要更换绑定的 OpenClaw 账号，需要重新执行安装流程并扫码绑定。形象地说："只能养一只虾，不能养多只虾"。

需要注意的是："不要误以为可以在多个微信号上同时使用同一个 OpenClaw"，正确认知是一对一绑定，换账号需重新扫码。

限制二：仅手机端支持

目前 ClawBot 只支持在手机版微信中使用，电脑版微信即使更新到最新版也没有 ClawBot 入口。这是因为该插件的设计初衷是让用户更方便地脱离电脑、在外出时也能灵活调用 OpenClaw。

限制三：默认绑定主 Agent，可切换

ClawBot 默认绑定的是你的主 Agent。如果要切换到其他 Agent，在终端执行：

openclaw config

记住："看到需要切换 Agent，立刻想到运行 openclaw config 重新绑定指定 Agent"。

限制四：官方安全隔离

作为官方插件，ClawBot 提供完整的安全隔离：他人无法添加你的 ClawBot，也无法将其拉进群聊。这是官方插件相对于第三方方案最重要的安全优势，可以理解为"私人管家，外人无法接触"。

【操作示例】ClawBot 限制理解与 Agent 切换

四项限制速记：单点绑定 · 仅手机端 · 可切 Agent · 官方安全

  1. 打开本地终端
  2. 执行：openclaw config
  3. 按提示将账号绑定到目标 Agent

结果：ClawBot 对话窗口中的 AI 身份切换为目标 Agent。

本课总结

本课总结

核心要点

  • 前置准备：微信升级到最新版本，确认"设置→插件"中出现 ClawBot 入口；若未出现，后台重启微信触发灰度推送
  • 安装绑定：终端执行 npx -y <官方命令> → 弹出二维码 → 用目标微信号扫码 → 手机端点击"连接"
  • 效果验证：先发"你好"测连通性，再发任务型消息验证 Agent 能力
  • 使用限制：单点绑定（一虾原则）、仅手机端、可通过 openclaw config 切换 Agent、官方安全隔离
  • 战略价值：AI 嵌入 IM 是主流趋势，越早上手先发优势越大

易错提醒

  • 不要在未看到 ClawBot 插件入口的情况下就开始安装步骤
  • 扫码时必须确认使用正确的微信号，因为绑定后无法在不重新扫码的情况下更换账号
  • 不要误以为可以在多个微信号上同时绑定同一个 OpenClaw（一对一绑定）
  • 电脑版微信不支持 ClawBot，不要在电脑端找入口
  • 不要使用非官方渠道的安装命令，应使用官方提供的完整命令

知识应用范围

本教程中学到的操作逻辑具有较强的迁移性：微信升级找灰度功能的方法，可迁移至其他 App 新功能激活场景；npx -y 命令行安装模式，可迁移至其他 Node.js 工具的安装；扫码授权绑定流程，是大多数"第三方服务绑定 IM 账号"场景的通用模式；ClawBot 的单点绑定与安全隔离设计，也是评估其他 AI-IM 集成方案时的重要参考维度。
